# 実装計画テンプレート

フェーズ別に分割し、以下のテンプレートに従う。

```markdown
## 実装計画

### Phase 0: 準備 (想定時間)

**目標**: [このフェーズのゴール]

**前提条件**: なし

タスク:
├── [ ] [タスク1]
├── [ ] [タスク2]
└── [ ] 動作確認

**成果物**:
- [作成/変更されるファイル一覧]

**完了条件**:
- [ ] 既存テストがパスする
- [ ] ビルドが成功する
- [ ] [追加の完了条件]

---

### Phase 1: [フェーズ名] (想定時間)

**目標**: [このフェーズのゴール]

**前提条件**: Phase 0 完了

タスク:
├── [ ] [タスク1]
├── [ ] [タスク2]
├── [ ] [タスク3]
└── [ ] 動作確認

**成果物**:
- [作成/変更されるファイル一覧]

**完了条件**:
- [ ] 既存テストがパスする
- [ ] 新規テストを追加し、パスする
- [ ] ビルドが成功する
- [ ] [追加の完了条件]

---

### Phase N: 仕上げ & テスト

**目標**: テスト追加、ドキュメント更新、最終確認

タスク:
├── [ ] テスト追加/更新
├── [ ] README.md 更新（必要な場合）
├── [ ] 不要コードのクリーンアップ
└── [ ] 最終動作確認

**完了条件**:
- [ ] 全テストがパスする (`npm run test:run`)
- [ ] ビルドが成功する (`npm run build`)
- [ ] Lint が通る (`npm run lint`)
```
