# 発火ポリシー

## 発火条件

次をすべて満たす場合にのみ、このスキルを発火する。

1. ユーザーが動画解析を明示的に依頼している。
2. 動画ファイルがある（または提供予定がある）。
3. 目的が動画証跡に基づく検証・切り分けである。

## 代表的な発火フレーズ

- 動画を解析して
- mp4 を見て確認して
- キャプチャ動画で不具合を調査して
- 動画ベースで状況を切り分けたい

## 発火しない条件

- 動画解析の明示依頼がない
- 動画証跡が不要で、コード対応のみで完結する

## 必須ガードレール

- 環境セットアップ（`npm run dev:media:setup` や依存インストール）前に、必ずユーザー確認を取る。
- ユーザー確認なしで依存導入を開始しない。
- 動画パスや目的が曖昧なら、先に最小限の確認を行う。
- まず実行経路を選ぶ（A: workspace helper 利用 / B: ポータブル venv 構築）。
- カスタムスクリプトは `scripts/dev` に限定し、CLI 引数で挙動を切り替えられる形にする。
- `scripts/dev` が無いワークスペースでは、必要時に `scripts/dev` を新規作成してから実装する。
