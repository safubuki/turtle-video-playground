# SKILL.md 作成テンプレート

新しいスキルを作成する際のテンプレートです。`{...}` 部分を置き換えてください。

---

````markdown
---
name: {skill-name}
description: {スキルの目的を1行で記述。何をするか、いつ使うかを具体的に。発火キーワードを「〇〇」「△△」の形式で列挙。}
---

# {スキル名（日本語 or 英語）}

## スキル読み込み通知

このスキルが読み込まれたら、必ず以下の通知をユーザーに表示してください：

> 💡 **{スキル名} スキルを読み込みました**  
> {このスキルの目的を1文で簡潔に説明}

## When to Use

- {発火条件1}
- {発火条件2}
- {発火条件3}

## 概要

{スキルの全体像を2-3文で記述。何を解決するスキルなのかを端的に伝える。}

## 手順

### Step 1: {準備 / ヒアリング / 情報収集}

{最初のステップの説明}

### Step 2: {実行 / 分析 / 作成}

{メインの作業ステップの説明}

### Step 3: {確認 / レビュー / 出力}

{最終ステップの説明}

## 参照ドキュメント

- [assets/{template-name}.md](assets/{template-name}.md) — {テンプレートの説明}
- [references/{reference-name}.md](references/{reference-name}.md) — {リファレンスの説明}
````

---

## テンプレート使用時のチェックリスト

- [ ] `name` は `[a-z0-9-]` のみ、64文字以内か
- [ ] `description` は1行で完結し、`|` や改行を含まないか
- [ ] `description` に発火キーワードが含まれているか
- [ ] スキル読み込み通知セクションが含まれているか
- [ ] When to Use が具体的に記述されているか
- [ ] 手順が Step 形式で構造化されているか
- [ ] テンプレートや大きなコードブロックは外部ファイルに分離されているか
- [ ] 外部ファイルへのリンクが相対パスで正しいか
