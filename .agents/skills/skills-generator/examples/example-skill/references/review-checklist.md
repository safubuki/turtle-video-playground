# コードレビュー詳細チェックリスト

## 品質
- [ ] 変数名・関数名が意図を適切に表しているか
- [ ] 関数の責務が単一か（Single Responsibility）
- [ ] 重複コードがないか（DRY原則）
- [ ] エラーハンドリングが適切か

## セキュリティ
- [ ] ユーザー入力のバリデーションがあるか
- [ ] 機密情報がハードコードされていないか
- [ ] SQLインジェクション/XSS等の対策があるか

## パフォーマンス
- [ ] 不要な計算や再レンダリングがないか
- [ ] 大量データの処理に問題がないか
- [ ] メモリリークの懸念がないか

## 保守性
- [ ] テストが書きやすい構造か
- [ ] コメントが適切か
- [ ] 複雑度が高すぎないか
